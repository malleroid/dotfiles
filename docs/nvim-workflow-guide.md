# Neovim ワークフローガイド

> Neovim + yazi + Zellij 環境での開発フローを、ユースケース別にステップで解説。
> 操作元を `[nvim]` `[yazi]` `[Zellij]` `[shell]` で明示。

---

## ツール分担の方針

| 役割 | ツール | 備考 |
|---|---|---|
| **テキスト検索・コード編集** | nvim (Telescope, LSP) | エディタの仕事 |
| **ファイル探索・ファイル操作** | yazi | コピー/移動/リネーム/削除・ディレクトリ探索 |
| **プロジェクト構造の俯瞰** | nvim (Neo-tree) | 今開いているファイルの周辺確認程度 |
| **ペイン管理・ターミナル並行** | Zellij | nvim + shell のレイアウト |

---

## 開発の起点は2パターン

### パターン A: 開きたいファイルがわかっている

```
[yazi] or [shell]          →  nvim でファイルを開く  →  編集開始
  ファイル名で探す                e キー or nvim <file>
  ディレクトリを掘る
```

1. `[yazi]` ディレクトリを移動してファイルを見つける → `e` で nvim が開く
2. または `[shell]` `nvim <ファイル名>` で直接開く
3. `[nvim]` 開いた後に関連ファイルも見たければ `<Space>ff` (Telescope) や `<Space>e` (Neo-tree) で辿る

### パターン B: テキスト検索から始める

```
[nvim]                     →  検索結果からジャンプ  →  編集開始
  <Space>fg (live grep)         Enter
  <Space>ff (ファイル名検索)
```

1. `[shell]` `nvim` を起動 (ファイル指定なしでもOK)
2. `[nvim]` `<Space>fg` でキーワード検索、または `<Space>ff` でファイル名検索
3. `[nvim]` 結果から `Enter` でジャンプ → そのまま編集

> **使い分けの目安**
> - ファイルパスやディレクトリ構造がなんとなくわかる → パターン A (yazi)
> - 関数名・エラーメッセージ・文字列など中身で探したい → パターン B (nvim)

## 1. ファイルを探して開く (nvim 内)

**場面**: プロジェクト内のファイルを名前で探してすばやく開きたい。

1. `[nvim]` `<Space>ff` — Telescope のファイル検索を開く
2. `[nvim]` ファイル名の一部をタイピング — リアルタイムで絞り込まれる
3. `[nvim]` `Ctrl+J` / `Ctrl+K` — 候補を上下移動
4. `[nvim]` `Enter` — 選択したファイルを開く

**開いたファイルの周辺を確認したい場合**:

5. `[nvim]` `<Space>e` — Neo-tree を開く (カレントファイルの位置に自動追従)
6. `[nvim]` Neo-tree 内で `j/k` で移動、`Enter` で開く

> **Tips**
> - Telescope の入力は fuzzy match なので `src/comp/btn` のような省略入力でも `src/components/Button.tsx` がヒットする
> - Neo-tree は開いたファイルを自動追従する (`follow_current_file`) ので、ディレクトリ構造の把握に便利

---

## 2. キーワードから目的のコードを見つける

**場面**: エラーメッセージや関数名で全文検索して、該当箇所にジャンプしたい。

1. `[nvim]` `<Space>fg` — Telescope の live grep を開く
2. `[nvim]` 検索したいキーワードを入力 — リアルタイムで全ファイルから検索
3. `[nvim]` `Ctrl+J` / `Ctrl+K` — 候補を上下移動 (ファイル名と行番号がプレビューされる)
4. `[nvim]` `Enter` — 該当行にジャンプ

**複数箇所を一括で確認したい場合**:

5. `[nvim]` 検索結果画面で `Ctrl+Q` — 全結果をクイックフィックスリストに送る
6. `[nvim]` `:cnext` / `:cprev` — クイックフィックスリスト内を順番に移動

> **Tips**
> - live grep は ripgrep ベースなので `.gitignore` で除外されたファイルは検索対象外
> - 検索結果が多すぎるときは、さらにパスやファイル名で絞り込む文字を追加する

---

## 3. コードを読み解く (LSP ナビゲーション)

**場面**: 知らないコードベースで関数の定義・利用箇所を辿りながら理解したい。

### 定義を辿る

1. `[nvim]` 関数名やクラス名の上にカーソルを置く
2. `[nvim]` `gd` — 定義へジャンプ
3. `[nvim]` `Ctrl+O` — 元の場所に戻る (ジャンプ履歴を遡る)

### 呼び出し元を調べる

1. `[nvim]` 関数名の上にカーソルを置く
2. `[nvim]` `gr` — 参照一覧が表示される
3. `[nvim]` 一覧から選んで `Enter` — その箇所にジャンプ

### インターフェース/型の実装を探す

1. `[nvim]` インターフェース名の上にカーソルを置く
2. `[nvim]` `gi` — 実装一覧へジャンプ

### その場で情報を確認

1. `[nvim]` 知りたいシンボルの上にカーソルを置く
2. `[nvim]` `K` — ホバードキュメント (型情報・JSDoc 等) がポップアップ表示

> **Tips**
> - `gd` → `gr` → `gd` をチェーンすることで、定義 → 利用箇所 → さらにその定義、と芋づる式にコードを辿れる
> - `Ctrl+O` (戻る) / `Ctrl+I` (進む) はジャンプ履歴のブラウザの「戻る/進む」ボタンに相当。深く潜っても迷子にならない

## 4. 複数ファイルを並べて編集する

**場面**: 実装ファイルとテストファイル、コンポーネントとその型定義など、関連ファイルを見比べながら作業したい。

### 方法 A: Neovim のウィンドウ分割

1. `[nvim]` 1つ目のファイルを開いた状態で `:vsplit` (縦分割) または `:split` (横分割)
2. `[nvim]` 分割先のウィンドウで `<Space>ff` — 2つ目のファイルを検索して開く
3. `[nvim]` `Ctrl+W` → `h/j/k/l` — ウィンドウ間のフォーカス移動

**ウィンドウ操作**:

| キー | 操作 |
|---|---|
| `:vsplit` or `:vs` | 縦分割 |
| `:split` or `:sp` | 横分割 |
| `Ctrl+W h/j/k/l` | ウィンドウ間移動 |
| `Ctrl+W =` | ウィンドウサイズ均等化 |
| `Ctrl+W _` | 現在のウィンドウを最大高さに |
| `Ctrl+W \|` | 現在のウィンドウを最大幅に |
| `Ctrl+W q` | ウィンドウを閉じる |
| `Ctrl+W o` | 他のウィンドウを全部閉じる |

### 方法 B: Zellij のペイン分割 + 各ペインで nvim

1. `[Zellij]` `Cmd+P` → `r` — 右に新規ペイン
2. `[Zellij]` 新しいペインで `nvim ファイル名` — 2つ目のファイルを開く
3. `[Zellij]` `Cmd+H/L` — ペイン間フォーカス移動

### どちらを使うか

| 観点 | nvim 分割 | Zellij ペイン |
|---|---|---|
| ファイル間の移動 | バッファ共有で高速 | 各ペインが独立 |
| LSP の共有 | 同一プロセスで全ウィンドウが共有 | ペインごとに別プロセス |
| ターミナル併用 | nvim 内に閉じる | シェルと nvim を自由に混在 |
| おすすめ用途 | 関連コードの同時閲覧・編集 | nvim + シェル (テスト実行等) |

> **Tips**
> - 基本は **nvim 分割で関連ファイルを並べ、Zellij ペインでシェルを横に置く** 組み合わせが便利
> - nvim のバッファは閉じても残っている。`<Space>fb` でバッファ一覧から素早く切り替えられる

## 5. Git 変更の確認・ステージ

**場面**: コードを書き終えた後、変更内容を確認してコミットしたい。

### 変更箇所の確認

1. `[nvim]` `]c` / `[c` — 変更箇所 (hunk) 間をジャンプ (サインカラムの `│` が目印)
2. `[nvim]` `<Space>hp` — 変更前との差分をプレビュー

### 選択的にステージ

3. `[nvim]` ステージしたい hunk の上で `<Space>hs` — その hunk だけステージ
4. `[nvim]` 不要な変更は `<Space>hr` — その hunk をリセット (変更前に戻す)
5. `[nvim]` ファイル全体をステージしたい場合は `<Space>hS`

### コミット

6. `[Zellij]` `Cmd+P` → `r` or `Cmd+N` — 隣のペインを開く (またはフォーカス移動)
7. `[Zellij]` シェルで `git status` → `git commit`

> **Tips**
> - `<Space>hb` で行ごとの blame (誰がいつ変更したか) を確認できる
> - `<Space>hd` でファイル全体の diff を表示
> - hunk 単位でのステージ (`<Space>hs`) を活用すると、1つのファイル内の変更を複数コミットに分けられる

## 6. ターミナル連携 (Zellij + Neovim)

**場面**: エディタでコードを書きながら、テスト実行やサーバー起動を並行したい。

### 基本レイアウト

```
┌──────────────────────┬─────────────────┐
│                      │                 │
│   nvim (メイン)       │   shell         │
│                      │  (テスト/サーバー) │
│                      │                 │
└──────────────────────┴─────────────────┘
```

1. `[Zellij]` Zellij 上で nvim を起動
2. `[Zellij]` `Cmd+P` → `r` — 右にシェルペインを作成
3. `[Zellij]` 右ペインでテストやサーバーを実行
4. `[Zellij]` `Cmd+H` / `Cmd+L` — nvim ⇔ シェル のフォーカス切替

### テスト実行のフロー

1. `[nvim]` コードを編集・保存 (`:w`)
2. `[Zellij]` `Cmd+L` — シェルペインにフォーカス
3. `[Zellij]` テストコマンドを実行 (例: `npm test`, `go test ./...`)
4. `[Zellij]` `Cmd+H` — nvim に戻って修正

### 複数プロセスの監視

```
┌──────────────────────┬─────────────────┐
│                      │  shell (テスト)   │
│   nvim (メイン)       ├─────────────────┤
│                      │  shell (サーバー)  │
└──────────────────────┴─────────────────┘
```

1. 上のレイアウトから `[Zellij]` 右ペインにフォーカスした状態で `Cmd+P` → `d` — 下に追加ペイン
2. 上ペインでテスト、下ペインでサーバー、のように使い分ける

> **Tips**
> - Zellij のペインはそれぞれ独立したシェルなので、nvim の保存を待たずにテストを回せる
> - `Cmd+P` → `f` でペインをフルスクリーンにすると、一時的に1ペインだけ広く使える (もう一度で戻る)
> - `Cmd+P` → `z` でペインのフレームを非表示にして画面を広く使える
