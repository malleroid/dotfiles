# NFR チェックリスト

要件構造化時に非機能要件の抜け漏れを防ぐためのチェックリスト。ISO 25010 (SQuaRE) をベースに実務で頻出する項目を整理。

## パフォーマンス効率

- [ ] 応答時間の目標値（p50/p95/p99）
- [ ] スループット要件（TPS、同時接続数）
- [ ] リソース使用量の制約（CPU/メモリ/ディスク）
- [ ] スケーラビリティ方針（水平/垂直、上限）

## 信頼性

- [ ] 可用性目標（SLA/SLO: 99.9% 等）
- [ ] RTO（目標復旧時間）
- [ ] RPO（目標復旧地点）
- [ ] データ整合性の方針（強整合性/結果整合性）
- [ ] バックアップ・リストア要件

## セキュリティ

- [ ] 認証方式（SSO、MFA、API キー等）
- [ ] 認可モデル（RBAC、ABAC、ACL）
- [ ] データ暗号化（保存時/転送時）
- [ ] 監査ログ要件
- [ ] コンプライアンス要件（GDPR、個人情報保護法等）
- [ ] 脆弱性管理方針

## 保守性

- [ ] コーディング規約・アーキテクチャパターン
- [ ] テスト戦略（カバレッジ目標、テスト種別）
- [ ] CI/CD 要件
- [ ] ドキュメント要件

## 運用性

- [ ] ログ要件（形式、保持期間、出力先）
- [ ] 監視・アラート要件
- [ ] デプロイ方式（Blue-Green、Canary 等）
- [ ] ロールバック手順
- [ ] 定期メンテナンスの許容ウィンドウ

## 互換性

- [ ] 対応ブラウザ / OS / デバイス
- [ ] 外部システムとの連携プロトコル
- [ ] API バージョニング方針
- [ ] データ移行要件

## ユーザビリティ

- [ ] アクセシビリティ要件（WCAG レベル）
- [ ] 多言語対応
- [ ] タイムゾーン・ロケール対応
- [ ] オフライン対応

## その他

- [ ] ライセンス制約
- [ ] 法規制・業界規制
- [ ] データ保持期間・削除ポリシー
- [ ] 災害復旧（DR）要件
