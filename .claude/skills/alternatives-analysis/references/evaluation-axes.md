# 評価軸マスターリスト

代替案を比較する際に使用する評価軸の候補。案件に応じて取捨選択・重み付けする。

## 技術的評価軸

| 評価軸 | 説明 | 確認ポイント例 |
|--------|------|---------------|
| パフォーマンス | レイテンシ・スループット・リソース効率 | ベンチマーク結果、理論値、実績値 |
| スケーラビリティ | 負荷増加への対応力 | 水平/垂直スケール、ステートレス性 |
| 信頼性・可用性 | 障害耐性・SLA 達成可能性 | SPOF の有無、冗長化方式 |
| セキュリティ | 脅威への耐性・コンプライアンス | 認証/認可、暗号化、脆弱性実績 |
| 保守性 | コードの理解・変更の容易さ | 複雑度、テスト容易性、ドキュメント |
| 拡張性 | 将来の機能追加への対応力 | プラグイン機構、API の柔軟性 |
| 相互運用性 | 既存システムとの統合の容易さ | プロトコル、データ形式、SDK |
| 成熟度 | 技術の安定性・実績 | バージョン、コミュニティ規模、バグ頻度 |

## ビジネス評価軸

| 評価軸 | 説明 | 確認ポイント例 |
|--------|------|---------------|
| 初期コスト | 導入・開発にかかるコスト | ライセンス費、開発工数、インフラ初期費 |
| 運用コスト | 継続的にかかるコスト | インフラ費、保守工数、ライセンス更新 |
| 開発速度 | 実装完了までの期間 | 学習コスト、エコシステム、ボイラープレート量 |
| チーム適合性 | チームのスキルセットとの一致 | 経験者数、学習曲線、採用市場 |
| ベンダーリスク | 特定ベンダーへの依存度 | ロックイン度、移行コスト、代替の存在 |
| ライセンス | 利用条件の制約 | OSS ライセンス種別、商用利用可否 |

## 評価スケール

| 記号 | 意味 |
|------|------|
| ◎ | 非常に優れている |
| ○ | 良い・十分 |
| △ | やや懸念・条件付き |
| ✗ | 不適・重大な懸念 |
| ? | 情報不足・要調査 |
