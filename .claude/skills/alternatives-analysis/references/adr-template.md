# ADR テンプレート

Architecture Decision Record のテンプレート。`alternatives-analysis` Skill の出力を ADR に転記しやすい形にするための参照。

## フォーマット

```markdown
# ADR-XXXX: <決定のタイトル>

## ステータス

提案 / 承認 / 却下 / 廃止 / 代替（ADR-YYYY に置換）

## コンテキスト

<決定が必要になった背景・課題・制約条件>

## 検討した選択肢

### 選択肢 A: <名称>

- 概要: <簡潔な説明>
- メリット: <利点>
- デメリット: <欠点>

### 選択肢 B: <名称>

- 概要: <簡潔な説明>
- メリット: <利点>
- デメリット: <欠点>

### 比較表

| 評価軸 | 重み | 選択肢 A | 選択肢 B |
|--------|------|---------|---------|
| ... | 高/中/低 | ◎/○/△/✗ | ◎/○/△/✗ |

## 決定

<選択した選択肢とその根拠>

## 影響

- <この決定により生じる影響・トレードオフ>
- <今後のアクションアイテム>
```

## 記載の原則

- **コンテキスト**: 決定に至った背景を、この ADR だけ読んでも理解できるように書く
- **選択肢**: 却下した案も含め、検討した全ての選択肢を記録する
- **決定**: 「何を選んだか」だけでなく「なぜ選んだか」を明記する
- **影響**: 技術的影響だけでなく、チーム・プロセスへの影響も記載する
