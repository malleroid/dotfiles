# Source spec: OpenAI API

- **単位**: 日付（1 日付 = 1 メッセージ）
- **ヘッダー**: `*OpenAI API* — <date> · <<url>|原文>`
- **`changes[]` の形**: `[<カテゴリ>] (<対象 API/モデル>) <本文>`。カテゴリ（Feature/Update/Fix/Announcement）と対象タグは**ソース側が付与したもの**で、分類のヒントとして使う（ただし最終判断はトピック規則が優先）。

## レンダリング規則

ヘッダーに続けて、`changes[]` を次の**トピック**へ分類し、この順で見出しを立てて列挙する（該当 0 件の見出しは省略）。**deprecation/retire が最上位**:

1. 🚨 **破壊的変更・deprecation・退役** — API/機能の廃止予告・shutdown 日・非互換変更。**shutdown/移行期限は太字で強調**。
2. 🆕 **新モデル** — GPT 系モデルのローンチ。capabilities は 1 項目に集約。
3. 🔧 **API 機能・パラメータ** — 新エンドポイント・新パラメータ・既存 API の機能追加
4. ☁️ **プラットフォーム展開** — Bedrock / Azure 等への展開、SDK、コンソール/プラットフォーム UI
5. 💰 **料金・課金・レート制限** — 課金方式の変更・値下げ/値上げ
6. 📝 **その他**

各行の書き方:

- 日本語タイトル（太字）＋ 1〜2 行の補足要約。対象 API/モデル名（`v1/responses` 等）は保持する。
- ソースカテゴリと配置トピックが食い違う場合はトピック規則を優先する（例: `[Update]` でも中身が deprecation 告知なら 🚨）。
