# Source spec: OpenAI Codex

- **対象**: Codex CLI / IDE 拡張 / Codex app / mobile (ChatGPT iOS・Android 内 Codex) / web
- **単位**: 日付（1 日付 = 1 メッセージ。同日複数エントリは 1 メッセージ内に併記）
- **ヘッダー**: `*OpenAI Codex* — <date> · <<url>|原文>`
- **`changes[]` の形**: エントリ単位のブロック。1 行目 `**タイトル** (topics)`、2 行目にエントリ個別のアンカー URL、以降は本文（`**New features**` 等のセクション見出し＋箇条書き）。

## レンダリング規則

ヘッダーに続けて、各エントリを**エントリごと**に出す（エントリ = リリース 1 つ。Claude Apps と同様のブロック形式）。エントリ内の箇条書きを次の**トピック**へ分類し、この順で並べ替える（該当 0 件の見出しは省略）:

1. 🔧 **機能** — 新機能・新コマンド・新設定・既存機能の能力追加
2. 🔁 **挙動変更** — 既存挙動 / デフォルトの変更
3. 🛡 **セキュリティ/権限** — 認証・アクセス制御・RBAC・サンドボックス
4. 💻 **プラットフォーム別** — 特定環境（CLI / IDE / app / mobile / Windows / SSH 等）限定の変更
5. ⚡ **パフォーマンス**
6. 🐛 **その他 bugfix** — 要約なしの 1 行訳で**全件列挙**（` / ` 区切り可）

各行の書き方:

- エントリのタイトルは日本語化して太字（バージョン番号・製品名は英語のまま）。タイトル直後にエントリ個別アンカーを `<URL|原文>` で付けてよい。
- トピック 1〜5 は日本語タイトル（太字）＋必要なら 1〜2 行要約。
- 原文の `**New features**` / `**Improvements and bug fixes**` セクションは分類ヒントとして使い、出力には残さない（トピック見出しに置き換える）。
