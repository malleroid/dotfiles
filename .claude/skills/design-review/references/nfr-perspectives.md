# 非機能要件（NFR）の観点

設計レビューで非機能要件を評価する際の参照リスト。ISO 25010 (SQuaRE) をベースに実務で頻出する項目を整理。

## パフォーマンス効率

| 項目 | 確認ポイント |
|------|-------------|
| 応答時間 | 主要操作の目標レスポンスタイム（p50/p95/p99）が定義されているか |
| スループット | ピーク時の処理件数・TPS が見積もられているか |
| リソース使用量 | CPU/メモリ/ディスク/ネットワーク帯域の見積もりがあるか |
| スケーラビリティ | 負荷増加時のスケール方針（水平/垂直）が決まっているか |

## 信頼性

| 項目 | 確認ポイント |
|------|-------------|
| 可用性 | SLA/SLO（例: 99.9%）が定義されているか |
| 耐障害性 | 単一障害点（SPOF）が排除されているか |
| 回復性 | RTO/RPO が定義され、復旧手順があるか |
| データ整合性 | トランザクション境界・結果整合性の方針があるか |

## セキュリティ

| 項目 | 確認ポイント |
|------|-------------|
| 機密性 | データ分類（公開/内部/秘密/極秘）と暗号化方針 |
| 完全性 | 改ざん検知・監査ログの設計 |
| 認証・認可 | 認証方式と最小権限の原則の適用 |
| 通信保護 | TLS/mTLS の適用範囲 |
| コンプライアンス | GDPR/個人情報保護法等の法規制への対応 |

## 保守性

| 項目 | 確認ポイント |
|------|-------------|
| モジュール性 | コンポーネント間の結合度と凝集度 |
| 変更容易性 | 変更の影響範囲が局所化されているか |
| テスト容易性 | 自動テストが書きやすい構造か |
| デプロイ容易性 | CI/CD パイプラインとの整合、ロールバック可能性 |

## 運用性

| 項目 | 確認ポイント |
|------|-------------|
| 可観測性 | ログ・メトリクス・トレースの3本柱が設計されているか |
| 構成管理 | 環境差分の管理方法（環境変数、設定ファイル等） |
| 自動化 | 定型運用作業の自動化方針 |
| インシデント対応 | アラート設計とランブックの計画 |

## 互換性

| 項目 | 確認ポイント |
|------|-------------|
| 相互運用性 | 外部システムとのプロトコル・データ形式の整合 |
| 後方互換性 | API・データスキーマの互換性維持方針 |
| 移行性 | 既存システムからの移行パスと並行稼働の計画 |

## ユーザビリティ（該当する場合）

| 項目 | 確認ポイント |
|------|-------------|
| アクセシビリティ | WCAG 準拠レベルの目標 |
| 国際化 | 多言語・多通貨・タイムゾーン対応 |
| エラー体験 | エラーメッセージの明確性とリカバリ導線 |
