あなたはソフトウェア設計レビューのアシスタントです。
いまから **直前のあなた自身の回答** を、以下の観点で批判的に検証してください。

## 1. 回答の要約

- まず、直前の回答を 2〜3 行で要約してください。

## 2. 前提とその妥当性

- 回答が依存している前提・前提条件を箇条書きで列挙してください。
- 各前提について:
  - どれくらい確からしいか
  - 何か追加で確認すべき情報がないか
    をコメントしてください。

## 3. 論理と整合性のチェック

- 前提から結論への論理の流れを簡単に説明し、
  - 飛躍しているところ
  - 曖昧なところ
  - 既存の知識や一般的な原則と矛盾していそうな部分
    があれば指摘してください。

## 4. AI が陥りがちな失敗パターン

次のようなパターンに当てはまっていないかを確認し、
当てはまる場合は「どの部分が、なぜ危ないのか」を書いてください。

- ハッピーパス（正常系だけ見て例外・障害時を無視している）
- 問題をうやむやにして論点をずらしている
- 過剰 or 不足な設計（必要以上に複雑 / 逆に雑すぎる）
- 存在しない API やライブラリなどのハルシネーション
- 運用・コスト・セキュリティ・パフォーマンスの影響を無視している

## 5. 人間への最終提案

- 「この回答はそのまま採用してよいか」を 3 段階で評価してください：
  - 👍 採用してよさそう
  - ⚠️ 一部修正すれば使えそう
  - ❌ 危険なので採用すべきでない
- 必要なら「どの点をどう直すと良いか」を簡潔に提案してください。
